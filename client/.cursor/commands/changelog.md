# 生成更新日志

根据 Git 提交历史生成 CHANGELOG。

## 指令

请根据最近的提交生成更新日志。

### Changelog 格式

```markdown
# 更新日志

## [1.2.0] - 2024-01-15

### ✨ 新功能
- feat(用户): 新增微信登录功能 (#123)
- feat(首页): 添加轮播图组件 (#125)

### 🐛 Bug 修复
- fix(订单): 修复支付状态显示错误 (#124)
- fix(表单): 修复验证码发送频率问题 (#126)

### 🔧 优化改进
- perf(列表): 优化长列表渲染性能
- refactor(API): 重构请求封装逻辑

### 📝 文档更新
- docs: 更新 README 安装说明
- docs: 添加组件使用文档

### 🔨 其他变更
- chore: 升级依赖版本
- style: 统一代码格式

---

## [1.1.0] - 2024-01-01

### ✨ 新功能
- ...
```

### 生成规则

1. 按版本号分组
2. 按类型分类（feat/fix/docs 等）
3. 关联 PR/Issue 编号
4. 包含作者信息（可选）

### 自动化

从 git log 提取信息：
- 提交信息
- 提交日期
- 作者
- 关联 Issue
