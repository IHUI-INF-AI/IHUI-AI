# 创建 Pull Request

生成专业的 PR 描述和变更说明。

## 指令

请根据当前分支的变更生成 Pull Request 描述。

### PR 模板

```markdown
## 📋 变更类型

- [ ] 新功能 (feat)
- [ ] Bug 修复 (fix)
- [ ] 代码重构 (refactor)
- [ ] 样式调整 (style)
- [ ] 文档更新 (docs)
- [ ] 测试相关 (test)
- [ ] 构建/部署 (chore)

## 📝 变更描述

### 改动概述
[简要描述这次 PR 的主要改动]

### 详细说明
- 改动点 1
- 改动点 2
- 改动点 3

## 🎯 关联 Issue

- Closes #xxx
- Related to #xxx

## 📸 截图/录屏

[如果是 UI 相关变更，请提供截图]

| 修改前 | 修改后 |
|--------|--------|
| [截图] | [截图] |

## ✅ 自测清单

- [ ] 本地构建通过
- [ ] 单元测试通过
- [ ] 功能测试通过
- [ ] 代码已自审
- [ ] 无控制台错误

## 📌 部署注意事项

[需要特别注意的部署事项，如环境变量、数据库迁移等]

## 🔍 Review 重点

[希望 Reviewer 重点关注的地方]
```

### 自动生成内容

- 从 git diff 分析变更文件
- 从 commit 信息提取改动点
- 识别变更类型
