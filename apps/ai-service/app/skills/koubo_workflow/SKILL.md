- 选题**只认 24 小时内首次出现的消息**；判定"新"看的是**消息原始首次发布时间**，不是"今天又看到"的转发时间。
- **旧消息当天重发/二次转发排除（明确·零容忍）**：24 小时窗口之外早已发布的消息，即便当天被某账号再次转发、二次推送，也**一律不算**"24 小时内的新消息"，**不得据此选题**。

# Koubo 口播稿完整工作流

## ⚠️ 约束优先写作法（v11.7 新增·杜绝迭代返工）

> **核心原则：写完才验 = 注定返工。先算预算再写 = 一遍过。**

### 写作前：每篇设定预算表

每篇动笔前先设定以下预算，用 `koubo_live_check.py` 验证通过后再写下一篇：

| 约束项 | 预算 | 检查方式 |
|--------|------|----------|
| 字数 | **475**（中点，留15字缓冲） | `koubo_live_check.py "正文"` |
| "我"次数 | **9**（留1个冗余） | 同上 |
| 句子长度 | **≤35字**（留7字安全边际） | 同上 |
| 二次冲击转折 | **必须1个在280-380区间** | 查看 `全部转折词位置` |
| 教练词 | **≥1个**（你不妨/你试试/你赶紧/现在就/先别） | 同上 |
| 收藏价值 | **≥5分**（每篇≥4分，8篇均分≥5） | 查看5维评分 |
| 转折词类型 | **8篇用不同转折词**（但/不过/然而/其实 各≤5篇） | `--quick` 跨篇模式 |
| 结尾类型 | **行动式:金句式:感慨式 各2-3篇** | 同上 |

### 写作中：实时自检

写完每篇正文后，**立即调用**：
```bash
python koubo_live_check.py "正文内容..."
```
看到 ✗ 就当场修，不等到8篇全写完再回头改。

### 写作后：一次性双门禁

8篇全部通过 `live_check` 后，跑：
```bash
python koubo_quality_gate.py MMDD.txt   # Gate①
python koubo_validate.py MMDD.txt        # Gate②
```
若还有 [!!]，用 `live_check` 定位单项→修复→重跑，不超过2轮。

### 归档收尾（工作流不可跳过硬步骤·2026-07-15 加固）

双门禁全绿 + 对话全文显示后，**必须**执行归档收尾，否则工作流视为未完成：
```bash
python archive_daily.py --finalize   # 当日 MMDD.txt 并入 历史口播稿汇编.txt 并清空 Output/
python archive_daily.py --check       # 校验当日稿已入汇编；返回0=完成，非0=立即重跑 --finalize
```
- ⚠️ 严禁只跑 `archive_daily.py`（默认模式只移"非当日"稿，当日稿会留在 Output 被误判为"已归档"）——这是已发生过的漏归档根因。
- ⚠️ 不建后台定时任务：归档是工作流内一步到位的收尾动作，与"今天文章发完了"等公众号指令无关。

### 0712迭代教训（永久记住）

1. **"可"字陷阱**：正文里"可"字当"可以"用不算转折词，但人眼容易误判
2. **差1字符**：A2的"但"在279位置，差1字符不进280-380区间
3. **加字=加句号**：补字数优先用短句+句号，不要堆长句
4. **收藏=关键词密度**：多嵌"步骤/方法/成本/避坑/工具"等词自然提分
5. **先预算后写**：不设预算直接写 = 必然返工（0712用了20+轮）
