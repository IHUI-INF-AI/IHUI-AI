## ★★ 写作第一原则：必须像真人深度思考手写（永久硬规则·最高优先级）

> **铁律：每一篇文章读起来必须像一个有血有肉的人，深度思考后亲手写出来的。不允许有一点AI写的味道和感觉。**

这不是"避免几个AI词"的问题，是整体质感的要求：

- **有"我"的存在感**：全文必须有作者本人的判断、体感、情绪。"我关注这个行业快3年了""说实话，这条新闻让我意识到""说点个人判断"——这些不是装饰，是灵魂。没有"我"的文章=AI文章。
- **口语化≠不专业**：说人话、像跟朋友聊天一样讲事情，但每句话背后有真实的信息量和思考深度。"翻译成大白话""打个比方""什么概念？"这些口语标记是真人写作的证据，必须保留。
- **有情绪波动**：真人写文章会激动、会意外、会扎心、会无奈。"没想到""让人意外的是""说实话我越看越觉得"——这些情绪标记让文章有温度。AI写不出真正的情绪。
- **有不完美的真实感**：真人写作会有口语化的转折（"但话说回来""不过有个问题"）、会有自我纠正（"等等，这里我说的不准确"）、会有主观偏见（"我个人更看好XX"）。太工整、太面面俱到、太客观中立=AI味道。
- **禁止"上帝视角"**：AI文章最大的特征是站在高处俯瞰一切、什么都知道、什么都分析到位。真人写作是从自己的视角出发，有认知边界，有"我不确定的地方"，有"我只说我知道的"。
- **info_card/建议段落**：口语化亲切的建议语气完全没问题，像朋友给你出主意一样写。但禁止"帮你XX"推销句式、禁止"避坑指南"营销标签、禁止"收藏起来"社交CTA——这些是触发词问题，不是语气问题。

**自检方法**：写完文章后，大声读一遍。如果听起来像一个人面对面跟你聊天、分享他的发现和判断——过关。如果听起来像一篇报告、一份分析、一个新闻稿——打回重写。

## ★ 受众对齐铁律（永久硬规则·最高优先级）

> **一句话判断标准：你妈/你对象/你学员家长能不能看懂、想不想转发？**

公众号文章的读者是**普通人**——上班族、学生、家长、退休老人、自由职业者。
**不是**程序员、**不是**投资人、**不是**AI从业者。

### 选题判断3秒法则
拿到任何选题，3秒内回答：
1. **这个选题跟普通人有什么关系？** → 答不上来就换选题
2. **普通人看到这个标题会不会点进来？** → 不会就换标题
3. **普通人看完会不会转发给朋友？** → 不会就改角度

### 选题红线（永久禁止）
- ❌ 纯技术拆解（向量数据库、推理加速框架、API架构设计）
- ❌ 纯开发者工具评测（IDE、编程框架、代码编辑器）
- ❌ 纯行业战略分析（融资、IPO、商业布局、生态位）
- ❌ 纯参数对比（模型跑分、吞吐量、延迟指标）

### 选题绿灯（鼓励方向）
- ✅ AI工具实操（普通人拿来就能用的）
- ✅ AI影响解读（对普通人工作/学习/生活/教育的影响）
- ✅ AI避坑指南（普通人容易踩的坑、容易交的智商税）
- ✅ AI+教育 / AI+副业/职场 / AI热点人话解读

### 角度转换公式
同一个热点，必须从普通人角度切入：
- ❌ "DSpark推理加速框架技术分析" → ✅ "AI回答越来越快了，背后原因跟你有什么关系"
- ❌ "Claude Code Hook玩法" → ✅ "我用AI每天省2小时，就靠这3个自动设置"
- ❌ "GPT-5.6的150万上下文" → ✅ "AI一次能读4本《三体》了，对你意味着什么"

### 与口播稿选题打通
- 每周选题统一规划：口播讲短视频版，公众号写同一话题的深度长文版
- 口播稿中数据好的话题（播放量>500），优先写成公众号长文
- 两者互相导流

## ★ 双模板工作流

> **核心：根据选题类型自动选择写作模板，不是所有文章都用同一个公式。**

### 模板选择决策树

```
拿到选题后，按顺序判断：

0. 【天花板评估】（不过不写）
   ├─ 受众面 ≤ C级 → 放弃（除非有S级事件加持）
   ├─ 受众面 B级 + 无具体新闻事件 → 放弃
   └─ 受众面 A/S级 或 B级+有事件 ↓

1. 有没有独家实测权限？
   ├─ 有 → 用【爆款实测模板】
   └─ 没有 ↓

2. 是不是大模型/AI工具的重大更新？
   ├─ 是 + 有对比数据 → 用【爆款实测模板】
   └─ 否 ↓

3. 是不是普通人关心的热点？
   ├─ 是 → 用【内容公式v2】
   └─ 否 → 放弃这个选题
```

### 两个模板的适用场景对比

| 维度 | 爆款实测模板 | 内容公式v2 |
|------|-------------|-----------|
| **选题类型** | 大模型更新/AI工具实测/国产vs海外 | 普通人痛点/热点解读/避坑指南 |
| **内容特点** | 有独家数据、跑分对比、价格表 | 有方法步骤、场景代入、行动建议 |
| **读者收获** | 选型决策依据 | 立刻能用的方法 |
| **篇幅** | 2000-3000字 | 1500-2500字 |

### 爆款实测模板的4个硬性条件（必须同时满足）
1. **信息稀缺性** — 灰度/内测/首发/独家权限
2. **自带热点词** — 产品名有搜索流量
3. **对比冲突** — 国产vs海外、新vs旧、便宜vs贵
4. **实用价值** — 读者看完能直接用

**任一条件不满足 → 退回用【内容公式v2】**

### 受众对齐铁律的例外说明
实测文章会包含技术参数和跑分数据，但通过以下方式保持可读性：每个数据绑定具体场景、用大白话解释术语、突出成本对比、给出明确选型建议。

**判断标准不变：** 普通人能不能看懂、想不想转发？

## ★★ 内容公式v2（核心策略）

> **一句话总结：从"新闻分析"转向"问题→解决"。你不是记者，你是朋友。**

### 爆款内容5步公式

1. **钩子（前3句）**：个人见闻或具体场景，让读者觉得"这说的就是我"
2. **问题**：这件事跟普通人有什么关系？影响你的钱包/工作/孩子/时间？
3. **解决**：具体方法、步骤、避坑指南——读者看完能立刻行动
4. **冲击**：真实数据、反转、意外发现——让人"哇"的瞬间
5. **收尾**：一句话总结 + 转发理由（"转给你那个正在XX的朋友"）

### 选题来源（优先级排序）
1. **抖音数据验证**：口播稿中播放量>500的话题 → 立刻写成公众号深度版
2. **普通人痛点**：赚钱/省钱/避坑/教育焦虑/求职/副业
3. **热点人话解读**：大事件用普通人能懂的话讲清楚 → 24小时内发
4. ~~纯技术分析~~ → **永久禁止**

### 选题天花板评估（动笔前第一关，不过不写）

**动笔前必须回答3个问题（30秒内）：**

**① 受众面有多宽？** S级(全人群) > A级(大群体) > B级(垂直群体) > C级(小众)。底线：B级以下不做。

**② 有没有具体新闻事件？** 有事件=有紧迫感=有流量。没有事件=流量封顶。

**③ 标题有没有"非点不可"的冲动？** 好文章标题≠爆款标题。爆款=具体事件+故事+情绪让人非点不可。

**⚠️ 铁律：受众面B + 无事件 = 不做。不管validate能拿多少分。**

### 写作前自检（动笔前30秒）
- [ ] 这个话题，我妈/我对象/我学员家长关心吗？
- [ ] 标题有具体数字吗？有情绪词吗？有好奇心缺口吗？
- [ ] 文章里有"我"吗？
- [ ] 读者看完能立刻做一件事吗？
- [ ] 读者会转发给谁？为什么？

### 流量满分预检（动笔前必做）

**① 品牌名预埋（流量预测·传播力）**
- validate.py规则：标题+前1000字内必须出现至少1个知名品牌名
- 品牌名清单：GPT / OpenAI / Claude / Anthropic / Google / Gemini / Apple / Meta / Microsoft / Nvidia / SpaceX / Tesla / 百度 / 阿里 / 腾讯 / 华为 / 字节
- lead段写完立即检查

**② 情感词分布（传播力·情感共鸣）**
- 全文需含≥3个情感冲击词：震惊 / 意外 / 没想到 / 真相 / 揭秘 / 竟然 / 居然 / 后背发凉 / 颠覆 / 反转
- 按section分布：lead段1个、第1章1个、第2-3章1个、结尾1个

**③ 身份词分布（传播力·身份信号）**
- 全文需含≥3种不同身份词：普通人 / 上班族 / 学生 / 家长 / 宝妈 / 打工人 / 毕业生 / 职场人 / 创业者 / 自由职业者

### AI味清零预检（写作中持续规避）

**AI味计算规则（validate.py，必须≤2%）：**
- 强AI词（句首，权重1.0）：需要指出的是 / 值得注意的是 / 综上所述 / 总而言之 / 换言之 / 不难发现 / 由此可见 / 从某种意义上说 / 毋庸置疑 / 显而易见
- 弱AI词（句首，权重0.3）：这一 / 这种 / 这样 / 这些
- 对称句式（权重0.3）：不仅是…更是 / 一方面…另一方面 / 既…又 / 不是…而是 / 首先…其次…最后 / 既…也

**写作铁律（3条禁令）：**
1. **禁止句首强AI词**：用"我认为""说白了""其实""问题是"等口语化过渡替代
2. **禁止句首弱AI词**：把"这些"换成具体名词，或调整语序让"这些"不出现在句首
3. **对称句式全文≤1处**：用"其实是…""关键在于…""问题出在…"等不对称表达

## ★★ 标题生成工作流

> **核心原则：生成≥5个候选标题，自动打分，选最优。**

### 生成流程
1. **确定核心卖点**：这篇文章最吸引人的1个信息点
2. **套用标题公式**（详见 `HOOKS-LIBRARY.md`），生成5个候选
3. **自动打分**：用validate.py的标题吸引力评分（0-10分）
4. **选最优**：取分数最高的标题

### 标题7要素检查（目标10/10）
- [ ] **具体数字(+2)**：标题必须含阿拉伯数字
- [ ] **情绪触发(+2)**：必须含情绪词（翻车/坑/骗局/扎心/打脸/真相/套路/陷阱/智商税等）
- [ ] **好奇心缺口(+2)**：问号/竟然/居然/才知道/秘密/但…发现/结果/其实/真相
- [ ] **对比反转(+1)**：但/却/竟然/居然/原来/结果/没想到
- [ ] **身份锚定(+1)**：普通人/上班族/学生/家长/打工人/毕业生
- [ ] **行动暗示(+1)**：方法/技巧/教程/指南/攻略/怎么/如何/避坑/省/赚
- [ ] **长度适中(+1)**：20-30字

**铁律：7项必须全部命中，缺任何一项标题打回重写。**

## ★★ 开头钩子工作流

> **核心原则：前200字必须命中≥5种钩子类型。详细模板见 `HOOKS-LIBRARY.md`。**

### 7种钩子类型（validate.py检测，每种+2分）
1. 个人故事：我(去/试/用/发现/看到)、(上周/昨天/前几天)
2. 对话引用：跟我说、(问/告诉/喊)我
3. 数据冲击：\d+[%％]、(暴涨/暴跌/翻倍/腰斩)、\d+倍
4. 直接提问：(你/大家)...(知道/有没有/想过/觉得)
5. 场景代入：(想象/假设/如果你/当你/每次)
6. 争议声明：(其实/事实上/说实话/直说了)、(很多人/大部分人)
7. 反常识：(不是/并非)...(而是/其实)、(反而/恰恰相反)

**铁律：前200字命中≥5种。问号+1分，对话引号+1分，上限10分。**

### 开头禁忌
- ❌ "随着AI技术的不断发展..." / "众所周知..." / "今天我们来聊聊..." / "XX公司近日发布了..."

### 开头信息密度上限
命中≥5种钩子时，每句话≤30字，钩子之间必须有叙事连接，不能硬拼。

## ★★ 流量优化工作流

> **公众号流量=曝光×点击×读完×互动。每一层都要优化。**

### 副标题/摘要
- ≤35字，1句话，必须有好奇心缺口
- 禁止关键词堆砌，必须用具体数据或反差制造悬念

### 爆款三要素

**要素1：情绪峰值设计（动笔前必做）** — 4个情绪锚点：冲击(开头)+扎心(中段)+顿悟(后段)+温度(结尾)。必须用具体的人和事承载。

**要素2：截图时刻（每篇≥1个）** — 放在highlight组件里，3条件至少满足2个：有数字、有态度、有身份。

**要素3：信息缺口（禁止模板化悬念桥）** — 禁止"不过XX还不是最X的"模板，必须用具体的人/事/数字制造好奇心。

### 结尾互动三件套（CSDN安全版）
1. **评论引导**：提具体问题（❌"评论区说说/聊聊/告诉我"）
2. **分享引导**：给具体分享理由（❌"收藏起来""转发给"）
3. **复访引导**：给下次来看的理由（❌"点个关注""下周"）

### 流量预测评分要素（validate.py第12项，≥6/10通过）

| 维度 | 分值 | 强制要求 |
|------|------|---------|
| 时效性 | 0-2分 | 前2段lead至少1个时效词 |
| 冲突性 | 0-3分 | 前2段lead至少1个冲突词 |
| 共鸣度 | 0-3分 | 前2段lead至少1个共鸣词 |
| 稀缺性 | 0-1分 | 选题不重复即可 |
| 传播力 | 0-1分 | 标题或前1000字必须有品牌名 |

**前2段lead最低要求**：时效性词≥1 + 冲突性词≥1 + 共鸣度词≥1 + 品牌名≥1 = 4个维度全覆盖

### ⚠️ validate满分 ≠ 流量保证
validate.py衡量"算法友好度"，不是"读者想不想点"。选题天花板在动笔前决定80%的流量上限。

## ★★ 选题验证工作流

> **核心原则：写完再发不如验证完再写。选题不对，写得再好也没人看。**

### 选题验证5步法

**第0步：天花板评估（30秒）**
- 受众面 ≥ B级 / 有具体新闻事件 / 标题有点击冲动
- **铁律：受众B + 无事件 = 不做**

**第1步：需求验证**
- 抖音搜索相关视频播放量TOP3 ≥ 1万
- 微信搜一搜有相关热搜词
- 知乎热榜关注>1000、回答>50
- **信源覆盖**：上述热榜均登记在 `lib/all_sources.py` 的 `cn_hot` 段（与口播稿共用同一份注册表），验证时直接按注册表逐源核对，避免漏源。

**第2步：差异化验证**
- 微信搜一搜看前10篇，角度不能雷同
- 别人都写跑分→你写价格算账；别人写技术→你写对普通人的影响

**第3步：时效性验证**
- 热点类：24小时内发；常青类：任何时间；周期类：匹配时间节点

**第4步：转发潜力验证**
- "读者看完会转发给谁？" 答案必须清晰具体

### 选题验证速查表

| 检查项 | 通过标准 |
|--------|---------|
| **受众面** | **≥ B级** |
| **事件驱动** | **有具体新闻事件** |
| **点击冲动** | **标题念出来朋友想追问** |
| 抖音相关视频播放量 | TOP3 ≥ 1万 |
| 同角度文章数量 | 前10篇中 ≥ 3篇角度不同 |
| 时效窗口 | 热点24h内 / 常青随时 |
| 转发对象明确 | 能说出具体身份 |

## 选题前置规则（永久硬规则）

**写公众号文章前必须先查 `公众号/wechat-article-system/已发布内容记忆.json`**：
1. 读取`published`数组，提取所有已发布文章的`title`、`topic`、`keywords`

#### 4a. 图片上传微信图床（发布前必做，不可跳过）

文章HTML中的本地图片在微信服务器上无法访问，**必须**先上传到微信图床获取CDN链接，再替换HTML中的本地路径：

```python
import sys, re, os; sys.path.insert(0, 'lib')
from wechat_publish import upload_image

html_path = 'output/文章.html'
with open(html_path, encoding='utf-8') as f:
    html = f.read()

# 提取所有本地图片路径
img_paths = re.findall(r'src="([^"]+)"', html)
url_map = {}
for local_path in img_paths:
    abs_path = os.path.join(os.path.dirname(os.path.abspath(html_path)), local_path)
    abs_path = os.path.normpath(abs_path)
    if os.path.exists(abs_path):
        wx_url = upload_image(abs_path)  # 返回URL字符串，不是dict
        if wx_url and isinstance(wx_url, str) and wx_url.startswith('http'):
            url_map[local_path] = wx_url

# 替换HTML
for local_path, wx_url in url_map.items():
    html = html.replace(local_path, wx_url)
with open(html_path, 'w', encoding='utf-8') as f:
    f.write(html)
```

> - `> 引用句` → 引用块（灰底`#F9FAFB`+虚线边框）

> **★★★ 固定发布工作流（唯一入口，禁止分步裸跑，2026-07-11 固化）★★★**
> 之前反复出错（样式不全/未满分就推送/残留Material绿/动画误解）的根因是"分步手工操作+靠自觉"。现已固化：

> - **门禁B（重点组件）**：渲染HTML重点颜色组件不足直接中止，逼作者在写作阶段就加标记
> - **草稿箱幂等**：先按标题删除旧草稿再创建，重复跑不会堆积重复草稿
> - **CSDN通用md导出**：流水线默认执行 `export_csdn_md.py` 生成 `{标题}.md`（公众号自定义语法→通用Markdown），供用户发CSDN；`--no-csdn` 可关
> - **预览动效（不留存 output）**：微信静态是平台限制（过滤`<style>`），浏览器看动效用 `gen_preview.py` 临时生成 `*_预览.html` 查看，**交付后即从 output 删除、不永久留存**，避免与 `{标题}.html` 形成"两个 html"混淆
> - **永久禁止**：自写 Material 绿（`#2e7d32`/`#1b5e20`/`#f1f8e9`）、自写极简HTML、跳过 validate 直接推送、使用 `validate_gzh_html.py`/`wrap_preview.py`/`md_to_docx.mjs`/`builder.py`/`csdn_publish.py` 等已废弃工具
>
> **★★★ Output 目录清理铁律（2026-07-11 用户第十三反馈"别留两个混淆"）★★★**
> 用户原话："html md格式文件为什么给我在Output里留存了两个 以后如果要生成另外一个 就要把之前历史无用的给删掉 别让我搞混"
> 每篇文章在 `output/` 只保留**一套清晰不重叠的交付物**，禁止留存历史无用版本让用户混淆：
> - **源 md 不在 output**：写作源/流水线输入 `{标题}.md` 统一放 `wechat-article-system/articles/` 目录（与交付物物理隔离，避免和 `{标题}{标题}.md` 形成"两个 md"混淆）
> - **output 只留「交付三件套 + images」**：`{标题}.html`(推送草稿箱) · `{标题}.docx`(文档交付) · `{标题}.md`(发CSDN) · `images/`(配图，推送后由微信URL替换本地路径)
> - **预览动效 html(`{标题}_预览.html`) 不永久留存于 output**：只是本地看动效的辅助文件，用 `gen_preview.py` 临时生成查看；交付完成后即从 output 删除，避免与 `{标题}.html` 形成"两个 html"
> - **生成新版本前先清旧版**：每次重新生成某产物（如重新跑流水线出新版 docx/html）前，先删除同标题旧的同名文件，绝不累积多版本
> - **严禁**在 output 留存"极简html/旧版html/调试草稿"等过渡文件；绝不允许出现两个同名 html 或两个重名 md 并存
