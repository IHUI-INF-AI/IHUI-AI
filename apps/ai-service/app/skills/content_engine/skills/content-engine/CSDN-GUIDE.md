# CSDN 适配规则（仅在发布到 CSDN 时加载）

> 本文件包含 CSDN 平台特有的改写工作流、广告防御规则和内容方向检测规则。
> **仅当文章需要同步发布到 CSDN 时才加载本文件，公众号发布时不需要。**
> 对应 SKILL.md v6.0 + REFERENCE.md §十二（跨平台复用）

---

## 一、CSDN高质量改写工作流

> **核心认知：公众号文章 ≠ CSDN文章。公众号写给人看（情绪+故事），CSDN写给技术人看（代码+数据+结构）。同一选题必须完全重写，不能直接搬运。**

### 1.1 CSDN质量分评分维度（目标≥80分）

| 维度 | 权重 | 核心要求 |
|------|------|----------|
| **内容深度** | 30-40% | 代码块+注释+算法分析+独特方案+实验数据 |
| **结构层级** | 25-30% | 清晰分级标题、逻辑递进、≥1500字 |
| **多元元素** | 20-30% | 代码块+图表占全文15-30%、中英文间加空格、外部引用链接 |

**加分项**：标题含技术关键词、代码注释详尽、多语言对比、实验数据、源码分析、性能测试
**扣分项**：标题党（震惊/不得不看）、结构混乱、无代码注释、图文比失衡、纯观点无数据

### 1.2 公众号→CSDN改写公式（6步转换法）

```
Step 1: 换标题 — 情绪词→技术关键词
  公众号: "GPT-5.6全面放开，普通人省钱的机会来了"
  CSDN版: "GPT-5.6三版本API实测：Sol/Terra/Luna性能对比与成本分析（附Python代码）"

Step 2: 换角度 — 行业分析→技术实操
  公众号: "这件事对普通人意味着什么"
  CSDN版: "用Python实测GPT-5.6三个版本的性能差异"

Step 3: 加代码 — 每1000字至少1个代码块（占比15-30%）
  必须包含：API调用示例、配置代码、benchmark脚本、数据处理代码
  代码必须有详细注释（每3-5行一个注释）

Step 4: 加数据表 — 用表格替代文字描述
  性能对比→结构化表格
  价格信息→价格对比表
  功能列表→功能对比矩阵

Step 5: 去营销化 — 删除所有公众号风格内容
  删除：所有emoji、互动CTA、"关注我"、情绪化感叹号
  替换：口语化表达→技术文档风格
  删除：geo_text中的公司全称（CSDN判为广告）
  保留：品牌简称"智汇AI"可保留1次

Step 6: 加技术深度 — 补充公众号版没有的内容
  添加：底层原理分析（模型架构、API设计思路）
  添加：环境配置教程（pip install、API key配置）
  添加：错误处理代码（异常捕获、重试机制）
  添加：性能测试数据（延迟、吞吐量、token消耗对比）
```

### 1.3 CSDN文章结构模板

```markdown
# [技术关键词] + [具体操作] + [量化结果]（附代码）

> 摘要：一段话说清楚本文做了什么、用了什么方法、得到什么结论。

## 一、背景与问题（是什么）
- 技术背景简述（2-3段，不要故事化）
- 本文要解决什么问题

## 二、环境准备（怎么做）
- 依赖安装：pip install xxx
- API配置/环境配置
- 版本信息

## 三、核心实现（代码+解析）
- 代码块1：基础调用示例（带详细注释）
- 代码块2：核心逻辑实现
- 代码块3：性能测试/benchmark脚本
- 每段代码后附2-3句解析

## 四、数据对比（表格+分析）
- 性能对比表（延迟/吞吐量/成本）
- 功能对比矩阵
- 关键数据用图表可视化

## 五、踩坑记录（实战经验）
- 遇到的问题+解决方案
- 版本兼容注意事项
- 性能优化技巧

## 六、总结
- 核心结论（3条以内）
- 适用场景建议
- 源码/数据下载链接（如有）

参考资料：
- [具体来源名称](URL) — 每条引用必须带链接
```

### 1.4 CSDN改写检查清单（build后逐项确认）

```
□ 代码块占比≥15%？（至少5个代码块，每个10-30行）
□ 每个代码块有注释？（每3-5行一个注释）
□ 有数据对比表格？（至少1个）
□ 标题含技术关键词？（模型名/API/框架名等）
□ 无营销词？（颠覆性/革命性/最强/神器等全部删除）
□ 无互动CTA？（评论区说说/聊聊/收藏/转发全部删除）
□ 无公司全称？（"吉林省爱智汇..."必须删除）
□ 有环境配置段落？（pip install / API配置）
□ 有错误处理代码？（try/except）
□ 参考资料带URL链接？（至少3条）
□ 中英文之间有空格？
□ 全文≥1500字且≤4000字？
□ 无"震惊""不得不看"等标题党？
```

### 1.5 实例：公众号→CSDN改写对照

| 维度 | 公众号版 | CSDN技术版 |
|------|---------|-----------|
| 标题 | GPT-5.6全面放开，普通人省钱的机会来了 | GPT-5.6三版本API实测：Sol/Terra/Luna性能对比与成本分析 |
| 开头 | "你有没有想过..." | "2026年7月8日，OpenAI正式发布GPT-5.6系列..." |
| 正文风格 | 口语化、比喻、情绪 | 技术文档、代码、数据表 |
| 代码 | 无 | Python SDK调用+benchmark脚本 |
| 数据呈现 | 文字描述 | 结构化表格+对比矩阵 |
| 结尾 | "评论区聊聊" | "参考资料+源码链接" |
| 字数 | 1500-3500 | 3000-4000（含代码） |

## 二、CSDN广告防御详细规则

> **核心认知：validate.py通过 ≠ CSDN通过。CSDN审核不仅看正则匹配，还做语义判断。**

### 2.1 validate.py内置CSDN_AD_PATTERNS（绝对禁止）

以下模式**绝对禁止**出现在文章任何位置：

**互动引流：**
- ❌ `点个关注` / `转发给XX朋友`
- ❌ `评论区说说/聊聊/告诉我`
- ❌ `收藏起来` / `收藏+下周/下期`

**机构宣传：**
- ❌ `本文作者为XX团队/公司/机构`
- ❌ `所属机构为XX公司/科技`
- ❌ `我们一直在帮助/服务/提供`

**跨平台引流：**
- ❌ `公众号/微信/抖音/小红书搜索/关注/扫码`
- ❌ `加我/添加/扫码/进群/入群`

### 2.2 CSDN语义广告触发防御（5条硬规则）

> **2026-07-08实测教训：validate 22项全过但CSDN判广告。根因：语义触发词。**

**规则1 — "普通人"频率上限**
- 全文"普通人"≤3次，"上班族"/"打工人"/"白领"/"宝妈"合计≤1次
- 替代方案：用"用户""使用者""从业者"替换

**规则2 — 触发词替换，语气不变**
- 口语化亲切语气完全没问题，CSDN判广告靠的是具体触发词
- 替换：`避坑指南`→`应对思路`、`帮你省钱`→直接陈述数据、`收藏起来`→删除
- info_card内容可保持对话式口吻，但不出现触发词

**规则3 — "避坑指南"等营销标签**
- ❌ `避坑指南`、`生存指南`、`自救手册`
- ✅ `应对思路`、`技术选型建议`、`使用指南`

**规则4 — "帮你省钱/帮你XX"句式**
- ❌ `帮你省钱`、`帮你提效`、`帮你避坑`
- ✅ 直接陈述事实（"成本降低96%"），让读者自己得出结论

**规则5 — "收藏/转发"类社交传播CTA**
- ❌ `收藏起来`、`记得收藏`、`转发给朋友`、`分享给XX`
- ✅ 直接删除，不加任何传播引导

### 2.3 build前CSDN触发词预扫（必须执行）

```python
import json, re
full_text = json.dumps(article, ensure_ascii=False)
csdn_triggers = {
    "普通人(>3次)": len(re.findall(r'普通人', full_text)),
    "上班族/打工人/白领": len(re.findall(r'上班族|打工人|白领|宝妈', full_text)),
    "帮你XX": re.findall(r'帮你\w{1,4}', full_text),
    "收藏": re.findall(r'收藏', full_text),
    "避坑": re.findall(r'避坑', full_text),
    "评论区说说/聊聊/告诉我": re.findall(r'评论区.{0,4}(说说|聊聊|告诉我)', full_text),
}
for k, v in csdn_triggers.items():
    if v and (isinstance(v, int) and v > 0 or isinstance(v, list) and len(v) > 0):
        print(f"  ⚠ CSDN触发: {k} = {v}")
```

### 2.4 CSDN安全替代方案

| 公众号表达 | CSDN安全替代 |
|-----------|-------------|
| "评论区说说/聊聊" | "你觉得XX怎么样？" / "你有类似的经历吗？" |
| "收藏起来方便以后翻看" | 删除 |
| "转发给正在找工作的朋友" | 删除 |
| "避坑指南" | "应对思路" / "使用注意事项" |
| "帮你省钱" | "成本降低96%"（直接陈述数据） |
| "点个关注" | 删除 |
| 公司全称"吉林省爱智汇..." | "智汇AI"品牌简称 |

### 2.5 品牌植入（CSDN安全方式）

- **禁止放在正文段落中**（会被判为软广）
- 以"作者注"形式出现在文章最末尾
- 禁止出现公司全称
- **作者注模板**：`（作者系智汇AI内容编辑，长期从事AI工具应用与就业信息整理工作。文中观点仅代表个人观察，不构成任何投资或就业建议。）`

### 2.6 结尾互动（CSDN安全版）

- ✅ "你觉得XX值不值得尝试？"（中性提问）
- ✅ "你有类似的经历吗？"
- ❌ "评论区说说/聊聊/告诉我"
- ❌ "收藏起来" / "点个关注" / "转发给"

## 三、CSDN内容方向检测

> **validate.py第20项检测，信息性预警（不阻断构建）**

### 3.1 分类规则
- 技术信号≥3个且多于非技术信号 → ✅ 适合CSDN
- 技术信号≥2个且≥非技术信号 → ⚠️ 需优化
- 其余 → ❌ 仅公众号

### 3.2 信号清单

**技术信号（正向）**：代码块/Python/JS/Go等语言名、API/SDK/框架/模型/算法、教程/实操/部署/配置/搭建、架构/源码/原理、GitHub/开源、Docker/Kubernetes/Linux、benchmark/性能/实测、pip install/npm install

**非技术信号（负向）**：职场/焦虑/鸡汤/励志、理财/基金/股票、普通人/上班族/打工人/白领、月薪/工资/收入、避坑指南/出路/自救、热搜/刷屏、收藏起来/分享给朋友

### 3.3 选题阶段决策
- 技术向选题（工具实测/代码教程/架构分析/数据爬取）→ 可同步CSDN
- 社会/职场/情感向选题 → 仅发公众号，不做CSDN适配

### 3.4 CSDN适配改造方法
如果选题偏社会向但需要发CSDN，必须加入至少3个技术信号：
1. 用Python/工具实测数据验证文中观点（代码+运行结果）
2. 拆解某个AI工具的技术实现原理
3. 提供可复现的操作教程

### 3.5 历史教训
- **2026-07-03**《AI每月吞掉1.1万岗位》：纯社会/职场向内容，CSDN拒收。原因：全文无代码、无工具实测、无技术分析
- **2026-07-08**《GPT-5.6政府批准全面放开》：validate全过但CSDN判广告。原因：geo_text含公司全称+评论区说说+"普通人"6次+"避坑指南"标题+"帮你省钱"句式
