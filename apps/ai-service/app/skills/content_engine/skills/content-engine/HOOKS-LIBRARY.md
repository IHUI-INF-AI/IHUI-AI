## 一、开头钩子模板库（统一版·7种）

> **核心原则：前200字必须命中≥5种钩子类型（满分10分=5种×2分）。每种+2分，问号+1分，对话引号+1分，上限10分。**

### 1. 个人故事（最推荐）

**公式：** `昨天/上周/前几天 + 我做了/看了/经历了什么 + 结果让我震惊/意外/后悔`

**validate.py检测正则：**
```
我(去|试|用|发现|看到|收到|经历|认识|心想|算了|扒了|查了)
(上周|昨天|前几天)
有(个|一位)(朋友|同事|学员|宝妈|大哥)
```

**示例：**
> "上周我让儿子用AI改了一周作业，英语成绩从72涨到89。但让我最意外的不是分数——是他改完作业后主动跟我说'妈，我再做一篇'。"

### 2. 数据冲击

**公式：** `一个让人惊讶的数字 + 这个数字意味着什么 + 跟读者有什么关系`

**validate.py检测正则：**
```
\d+[%％]
(暴涨|暴跌|翻倍|腰斩)
\d+倍
(过万|不到)\d+
```

**示例：**
> "智联招聘最新数据：2026年Q1，AI相关岗位平均薪资比传统岗位高47%。但更扎心的是另一个数字——83%的求职者根本不知道这些岗位的存在。"

### 3. 直接提问

**公式：** `一个读者会在心里回答"是"或"不是"的问题 + 转折/意外答案`

**validate.py检测正则：**
```
(你|大家)...(知道|有没有|想过|觉得)
(为什么|是不是|难道)
(怎么办|好不好)
```

**示例：**
> "你每天花多长时间刷短视频？如果超过2小时，接下来的内容可能改变你的职业选择。"

**注意：这是最容易漏掉的钩子类型之一——标题后的第一句就用问号。**

### 4. 争议声明

**公式：** `一个跟大众认知相反的判断 + 简短理由`

**validate.py检测正则：**
```
(其实|事实上|说实话|直说了)
(很多人|大部分人)...(都|是|以为)
(真相|事实)
被(忽悠|骗|误导)
```

**示例：**
> "我直说了：90%的人学AI都学错了方向。不是学太多，是学太少——而且学的都是最不值钱的部分。"

### 5. 场景代入

**公式：** `描述一个读者正在经历或即将经历的场景 + 暗示有更好的方式`

**validate.py检测正则：**
```
(想象|假设|如果你|当你|每次)
(你正在|你试过|你有没有过)
```

**示例：**
> "每次写周报都花2小时？每次做PPT都加班到10点？我找到一种方法，同样的工作只需要20分钟。"

**注意：这是第二容易漏掉的钩子类型——"你有没有过...""想象一下..."**

### 6. 对话引用

**公式：** `一句真实的对话/评论 + 引发思考的转折`

**validate.py检测正则：**
```
跟我说
(问|告诉|喊|叫)我
朋友(给|发|转)
(她|他)(说|问|觉得)
```

**示例：**
> "昨天一个家长问我：'AI会不会让我的孩子失业？'我说：'不会。但不会用AI的孩子，会被会用AI的孩子淘汰。'"

### 7. 反常识

**公式：** `一个违反直觉的事实 + 解释为什么`

**validate.py检测正则：**
```
(不是|并非)...(而是|其实)
(反而|恰恰相反)
跟...(想的|以为的)...不一样
```

**示例：**
> "花72块用Claude的人竟然被监控了？我扒完代码发现3个国产替代更香。"

### 组合模式（冲10/10分的秘诀）

单一钩子模式最多得2-4分。**满分开头必须组合3种以上钩子**：

| 组合 | 公式 | 预期分 |
|------|------|--------|
| 故事+数据+提问 | 昨天我经历了XX + 数据冲击 + 你敢信吗？ | 8-9/10 |
| 故事+数据+反常识 | 我扒了数据 + 差距XX倍 + 同样XX差了XX | 9-10/10 |
| 对话+争议+场景 | 一句真实对话 + 反常识判断 + 你是不是也这样 | 8-10/10 |
| 场景+故事+对话+数据+争议 | 全组合 | 10/10 |

**validate.py检测的7种钩子类型**：个人故事、对话引用、数据冲击、直接提问、场景代入、争议声明、反常识。每种+2分，问号+1分，对话式+1分，上限10分。**目标：前200字内命中≥5种。**

### 满分示例

> "你有没有算过自己每个月在AI工具上花了多少冤枉钱？上周我帮一个朋友查账单，打开一看我直接傻眼了——72块。什么概念？同样的活国产工具11块就搞定了。说实话，很多人根本不知道..." → 10/10（场景代入+个人故事+对话+数据冲击+争议声明+问号+反常识）

### 开头禁忌（绝对不要用）
- ❌ "随着AI技术的不断发展..." → 读者直接划走
- ❌ "众所周知，AI正在改变..." → 像教科书
- ❌ "今天我们来聊聊..." → 像老师讲课
- ❌ "最近，XX公司发布了..." → 像新闻稿

## 二、爆款实测开头4种模式

> **实测类文章专用开头模式，对应 REFERENCE.md §一。**

### 模式A：insider视角（最推荐）
```
[产品名][动作]跑了快一周，圈内[人群]基本都在私下传[结果]。
没有[常规宣传方式]，没有[常规宣传方式]，就靠[稀缺获取方式]，口碑反而越传越炸。

我找[关系]借了[稀缺资源]，连续跑了[时间]实测，覆盖[场景1]、[场景2]、[场景3]。
结论很明确：这不是[贬低描述]，是把[旧版本]最被吐槽的[问题]，一次性给补平了。
```

### 模式B：数据冲击开头
```
一个让人震惊的数字 + 这个数字的来源 + 跟竞品的差距 + 这意味着什么

示例：
"94.2分。这是DeepSeek V4.1在LiveCodeBench上的代码得分。
什么概念？它超过了Claude Opus 4.6的88.8分，和GPT-5.5只差不到2分。
半年前，没人敢信国产模型能在代码基准上追平海外第一梯队。"
```

### 模式C：反常识声明开头
```
一个跟大众认知相反的判断 + 简短理由 + 悬念

示例：
"我直说了：现在花200块/月用GPT的人，大部分都在白花钱。
不是GPT不好，是DeepSeek V4.1已经能做到90%的体验，价格只有1/36。
我连续跑了三天实测，结论可能让很多人不舒服。"
```

### 模式D：场景代入开头
```
描述一个读者正在经历的具体痛点 + 暗示有解决方案 + 悬念

示例：
"每次让AI读长文档，它总是在最关键的地方'失忆'。
问它第200页的参数定义，它给你编一个答案。
我测了6个大模型的长文本能力，有一个的召回率从83%飙到了91%。"
```

|------|------|------|----------|
| A | [产品]+[独家动作]：[解决痛点]，[大胆结论] | DeepSeek V4.1灰度实测：补完最后两块短板，国产模型真的能平替海外旗舰了 | 有独家权限时 |

| E | [产品]悄悄[动作]，[人群]已经[结果] | DeepSeek V4悄悄开放灰度，开发者圈已经炸了 | 信息稀缺时 |

| H | [时间]前[旧状态]，[时间]后[新状态]：[产品]做到了 | 半年前不敢想，今天它做到了：国产模型代码跑分首超GPT | 进步巨大时 |
| I | [产品]发布，我算了一笔账：[数字冲击] | DeepSeek V4发布，我算了一笔账：年省20万 | 成本角度 |
| J | 别再[旧做法]了，[产品]已经[新能力] | 别再花冤枉钱买GPT了，DeepSeek V4只要1/36的价格 | 替代类 |

### 标题7要素检查（目标10/10）

| 要素 | 分值 | 检查标准 |
|------|------|---------|
| 具体数字 | +2 | 标题必须含阿拉伯数字（"6倍""72块""3个"） |
| 情绪触发 | +2 | 含情绪词：翻车/坑/骗局/扎心/打脸/真相/套路/陷阱/智商税/暴雷/傻眼/懵了 |
| 好奇心缺口 | +2 | 问号"？"/竟然/居然/原来/才知道/秘密/但...发现/结果/其实/真相/没想到/我...扒完/算完/实测/亲测 |
| 对比反转 | +1 | 但/却/竟然/居然/原来/结果/没想到 |
| 身份锚定 | +1 | 普通人/上班族/学生/家长/打工人/毕业生 |
| 行动暗示 | +1 | 方法/技巧/教程/指南/攻略/怎么/如何/避坑/省/赚 |
| 长度适中 | +1 | 20-30字 |

**铁律：7项全部命中才通过。候选≥5个，逐一打分，选最高分。**

### 标题反面教材
- ❌ "Claude翻车：贵6倍效果还差，普通人别再花冤枉钱了" → 6/10（缺好奇心缺口、对比反转、行动暗示）
- ✅ "花72块用Claude的人竟然被监控了？我扒完代码发现3个国产替代更香" → 10/10

### 标题生成流程
```
Step 1: 确定核心卖点（只能1个）
Step 2: 套用10种公式，生成5-10个候选标题
Step 3: 用7要素逐一打分
Step 4: 选最高分的标题
Step 5: 最高分<10 → 继续改直到有10分候选
```

## 五、爆款标题铁律（详细版）

> **核心改变：不再强制要求产品名/品牌名。标题必须通过「点击冲动测试」。**

### 3个条件（全部通过）

**条件1（点击冲动）**：标题必须同时具备以下至少2项：
- 具体数字（"200""8小时""3个坑"等）
- 情绪触发词（"坑""骗局""扎心""翻车""套路""智商税"等）
- 好奇心缺口（"但我扒完发现""没想到""原来""竟然"等）

**条件2（具体利益）**：包含普通人利益点（省钱/省时间/避坑/赚了）
- 必须是**具体利益点**
- **禁止空洞否定收尾**

**条件3（长度）**：≤35字通过，36-45字警告，>45字强制不通过

### 公式
`[数字/情绪词] + [好奇心缺口] + [普通人具体利益]`

### 正确示例
- ✅ "日薪200给机器人当老师？但我扒完发现坑比想的多"
- ✅ "豆包突然开始收费了，但这3个免费功能普通人反而用得更爽"
- ✅ "我用AI帮孩子改了3天作业，发现一个很多家长不知道的坑"
- ✅ "68块钱一个月的AI，我用了7天，发现最值钱的功能不是你想的那样"

### 错误示例
- ❌ "具身智能数据采集员岗位深度分析"（无数字无情绪无好奇心缺口）
- ❌ "豆包专业版开闸：3.36亿月活、三级定价、编程追平Opus 4.7"（缺冲突+缺普通人利益）
