二、增强开发详情

1. Checkpoint 快照与回滚系统 新增

对标: Aider (Git auto-commit + revert) / Gemini CLI (Checkpointing)

文件: server/app/api/v1/workspace/checkpoint.py (新建, 460+ 行)

能力:

  • snapshot_before_modify() — 在 write/edit/delete 前自动快照原始文件内容到磁盘
  • undo_last() — 撤销最近一次修改, 恢复文件到快照状态 (含撤销新建文件)
  • rollback_to(checkpoint_id) — 回滚到指定检查点, 撤销该点及之后所有修改
  • list_checkpoints() — 列出检查点历史 (工具名/文件/状态)
  • 内存 + 磁盘双存储, 按工作区隔离, FIFO 淘汰 (最多 100 个)
  • 边界处理: 二进制文件、新建文件、备份丢失均有降级逻辑

REST API: GET /checkpoints / POST /checkpoints/undo / POST /checkpoints/rollback / GET /checkpoints/{id} / DELETE /checkpoints

2. Diff 预览 + dry_run 模式 增强

对标: Claude Code diff 预览 / Aider SEARCH/REPLACE 可视化

文件: server/app/api/v1/workspace/tools.py

能力:

  • write_file / edit_file / multi_edit 新增 dry_run 参数
  • dry_run=true 时仅返回 unified diff 预览, 不实际写入文件
  • 实际写入时也返回 diff 摘要, 帮助 Agent 理解变更范围
  • _generate_unified_diff() 使用 Python difflib, 支持 200 行截断

3. 模糊匹配恢复 增强

对标: Claude Code edit 工具容错能力

文件: server/app/api/v1/workspace/tools.py

能力:

  • _fuzzy_find_match() — 当 old_text 精确匹配失败时, 归一化空白后重新匹配
  • 处理制表符 vs 空格、多余空行、尾部空白等常见差异
  • edit_file 和 multi_edit 自动触发模糊匹配, 返回提示 "(模糊匹配恢复成功)"
  • 仍保证唯一性要求, 避免误替换

4. 并行工具执行 增强

对标: Claude Code / Codex 并发工具调用

文件: server/app/api/v1/workspace/agent_loop.py

能力:

  • 当一轮中所有工具调用都是只读工具 (read_file/grep/glob/git_*/web_*/todo_read/list_checkpoints) 时, 使用 asyncio.gather 并行执行
  • 权限检查 + PreToolUse Hook + 工具执行 + PostToolUse Hook 全部并行
  • return_exceptions=True 确保单个工具异常不影响其他工具
  • 事件推送和 messages 追加保持顺序一致性
  • 写工具 (write/edit/delete/run_command) 仍顺序执行, 保证安全

5. LLM API 重试退避 增强

对标: Codex Rust 稳健容错 / Gemini CLI 容错机制

文件: server/app/api/v1/workspace/llm_gateway.py

能力:

  • chat_with_tools() 新增 max_retries=3 参数, 指数退避 (1s → 2s → 4s)
  • 在流式输出开始前遇到可重试错误 (超时/连接/429/5xx) 自动重试
  • 流式输出开始后不重试, 避免重复输出
  • _is_retriable_error() 支持中英文错误关键词识别 (超时/连接/限流/过载/临时/重试)
  • 底层 httpx.TimeoutException / ConnectError / ReadError 也纳入重试

7. 智能上下文压缩 增强

对标: Claude Code auto-compact / Codex context window management

文件: server/app/api/v1/workspace/agent_loop.py

能力:

  • 保留尾部消息从 10 条增加到 12 条

8. Git Auto-Commit 双模式 新增

对标: Aider (Git auto-commit + revert) / Claude Code (Git 原生集成)

文件: server/app/api/v1/workspace/checkpoint.py

能力:

  • 双模式通过环境变量 IHUI_GIT_AUTOCOMMIT 控制:
  • IHUI_GIT_AUTOCOMMIT=1: Aider 风格 — git add + git commit (真实提交到历史, 可 git revert)
  • 默认 (未设置): 轻量模式 — git stash create (不污染提交历史)
  • _git_auto_commit() — 精准 add 被修改文件, commit 消息含检查点 ID 关联
  • _git_auto_commit_after_undo() — 撤销/回滚后自动提交 revert 记录
  • 非 git 仓库 / 无修改 / 进程异常均静默降级, 不影响主流程
  • commit 消息格式: feat(agent): {tool} [{checkpoint_id}] / revert(agent): {action} [{checkpoint_id}]

CLI 集成: /tools 命令输出环境变量说明; 系统提示词包含 Git 集成使用指导
