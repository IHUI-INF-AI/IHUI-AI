> 维护日期：2026-06-24
> 适用对象：运维 DBA、数据迁移工程师
> 范围：H 盘 22 个 Java 微服务的 MySQL 业务库 → G 盘 PostgreSQL `zhs_platform` 库
> 风险等级：🔴 P0（数据迁移是上线前最关键一步）

## 一、迁移核心难点

### 1.1 主键类型不兼容（最关键）

| 来源 | 主键类型 | 示例 |
|---|---|---|
| H 盘 MySQL | `BIGINT` 自增 | `t_member.id = 12345` |
| G 盘 PostgreSQL | `VARCHAR(64)` UUID | `users.uuid = 'a1b2c3d4...'` |

**直接 ETL 会导致：**
- 所有外键关联断裂
- 业务查询 `WHERE user_id = ?` 无法命中
- 历史评论/订单/支付流水无法关联回原用户

### 1.2 业务字段类型差异

| H 字段 | H 类型 | G 字段 | G 类型 | 影响 |
|---|---|---|---|---|
| 金额 (price, totalAmount) | `DECIMAL(10,2)` | `BIGINT` (分) | 整数 | 单位转换（×100） |
| 状态 (status) | `TINYINT` 枚举 | `INTEGER` 或 `VARCHAR(20)` | 枚举值不变 | 业务层读取需确认 |
| 创建时间 | `DATETIME` | `TIMESTAMP` | 毫秒精度 | 一般兼容 |
| 文本 (description) | `VARCHAR(2000)` | `TEXT` | 不限长 | 长度限制解除 |

### 1.3 业务表拆分差异

- H 盘 `t_member_company_member_relation.member_id` 是 BIGINT
- G 盘 `edu_member_company_member_relation.member_id` 是 VARCHAR(64) UUID
- 需要"主键映射中间表"

## 二、解决方案：id_mapping 主键映射表

### 2.1 表结构

见 [`server/app/models/id_mapping.py`](file:///g:/IHUI-AI/server/app/models/id_mapping.py)：

```sql
CREATE TABLE id_mapping (
    id              VARCHAR(64)   PRIMARY KEY,
    source_table    VARCHAR(64)   NOT NULL,   -- H 盘 t_xxx
    old_id          VARCHAR(32)   NOT NULL,   -- H 盘 BIGINT (字符串形式防精度)
    new_uuid        VARCHAR(64)   NOT NULL,   -- G 盘 UUID
    migration_batch VARCHAR(32)   NOT NULL,   -- 批次号 v2026_06_24_xxx
    remark          VARCHAR(500),
    created_at      TIMESTAMP     DEFAULT NOW(),
    updated_at      TIMESTAMP     DEFAULT NOW(),
    UNIQUE INDEX uk_idm_source_old (source_table, old_id),
    INDEX idx_idm_source_new (source_table, new_uuid),
    INDEX idx_idm_batch (migration_batch)
);
```

### 2.2 使用流程

```python
from app.models.id_mapping import IdMapping
from app.database import get_session

def get_new_id(source_table: str, old_id: int) -> str:
    """根据 H 盘老主键查 G 盘新主键."""
    with get_session() as db:
        m = db.query(IdMapping).filter(
            IdMapping.source_table == source_table,
            IdMapping.old_id == str(old_id),
        ).first()
        return m.new_uuid if m else None

def record_mapping(source_table: str, old_id: int, new_uuid: str, batch: str):
    """插入一条映射记录."""
    with get_session() as db:
        db.add(IdMapping(
            id=new_uuid,  # 主键复用 new_uuid
            source_table=source_table,
            old_id=str(old_id),
            new_uuid=new_uuid,
            migration_batch=batch,
        ))
        db.commit()
```

## 三、迁移执行步骤

### 3.1 预检查

```bash
# 1. 确认 H 盘 22 个微服务的 MySQL 库可访问
mysql -h h-mysql-host -u readonly -p -e "SHOW DATABASES"

# 2. 确认 G 盘 PostgreSQL 库已就绪
psql -h pg-host -U zhs -d zhs_ai_project -c "\dt"

# 3. 确认 DDL 已生成
ls -la g:\IHUI-AI\server\alembic\versions\001_init.sql   # 236 张表

# 4. 应用 DDL
psql -h pg-host -U zhs -d zhs_ai_project -f 001_init.sql
psql -h pg-host -U zhs -d zhs_center_project -f 001_init.sql
psql -h pg-host -U zhs -d zhs_educational_training -f 001_init.sql
```

### 3.2 分批迁移（按业务域）

```bash
# 批次号格式: vYYYY_MM_DD_XX
# v2026_06_24_01 会员 (edu_member 13 张表)
# v2026_06_24_02 课程 (t_lesson 等 25 张表)
# v2026_06_24_03 考试 (exam 等 19 张表)
# v2026_06_24_04 问答 + 圈子 (ask + circle 等)
# v2026_06_24_05 支付 + 订单 (edu_payment 等 6 张表)

# 每批次执行: 抽取 → 转换 (主键映射+单位转换) → 装载 → 校验
python scripts/migrate.py --batch v2026_06_24_01
```

### 3.3 校验

```sql
-- 行数对比
SELECT
    (SELECT COUNT(*) FROM t_member) AS h_count,
    (SELECT COUNT(*) FROM edu_member) AS g_count;

-- 抽样校验
SELECT m.old_id, m.new_uuid
FROM id_mapping m
WHERE m.source_table = 't_member'
LIMIT 10;

-- 一致性校验: 业务表外键是否都映射到 G 盘
SELECT COUNT(*) AS unmapped
FROM t_sign_up su
LEFT JOIN id_mapping m
  ON m.source_table = 't_member' AND m.old_id = su.member_id
WHERE m.id IS NULL;
```

### 3.4 回滚

```sql
-- 删除当批次插入的 G 盘数据
DELETE FROM edu_member WHERE created_at >= '2026-06-24 00:00:00';
-- 删除当批次映射
DELETE FROM id_mapping WHERE migration_batch = 'v2026_06_24_01';
```

## 四、迁移脚本框架（待实现）

`server/scripts/migrate.py` 框架：

```python
"""数据迁移 CLI 入口 (待补全)."""
import argparse
from typing import Iterable

BATCHES = {
    "v2026_06_24_01": {
        "tables": ["t_member", "t_member_company", "t_member_tag"],
        "target": "edu_member",
    },
    # ... 其它批次
}

def migrate_batch(batch_name: str):
    cfg = BATCHES[batch_name]
    for src_table in cfg["tables"]:
        extract(src_table, batch_name)
        transform(src_table, batch_name)  # 主键映射 + 单位转换
        load(src_table, cfg["target"], batch_name)
    verify(batch_name)


if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--batch", required=True, choices=BATCHES.keys())
    args = parser.parse_args()
    migrate_batch(args.batch)
```

> **注意**：本框架为占位脚本，实际 ETL 实现属于"新增功能"范围，按用户规则需要单独确认才能开发。

## 五、迁移风险与监控

| 风险 | 等级 | 应对 |
|---|---|---|
| H 盘主库写入未停 | 🔴 P0 | 迁移前停写或 binlog 增量订阅 |
| 主键冲突 | 🟠 P1 | id_mapping 唯一索引兜底 |
| 字符集不一致 | 🟠 P1 | H 盘 utf8mb4 → G 盘 UTF8 |
| 软删除数据 | 🟡 P2 | 确认 deleted=1 是否需要同步 |
| 大表 (t_record) | 🟠 P1 | 分页 1000 行/批 |
| 跨服务外键 | 🟠 P1 | 按依赖顺序：member → lesson → signup → record |
