## 2. 部署步骤 (运维执行)

### 2.1 部署前预检 (P0)

```bash
# 1. 拉取代码
cd g:\IHUI-AI
git pull

# 2. 检查 alembic head 是否到达 047
cd server
alembic current
# 期望: 047_notify_persist (or 后续 048...)

# 3. 若 head 落后, 补跑迁移
alembic upgrade head
```

### 2.2 配置环境变量 (.env)

确保生产 `.env` 含以下配置 (若缺失则补):

```ini
# 站内信 (P1 封版 047) — 多副本必须一致
NOTIFY_RECIPIENT_UUID=00000000-0000-0000-0000-000000000001
NOTIFY_MAX=1000

# Celery (beat + worker) — 必须与后端共用同一 Redis
CELERY_BROKER_URL=redis://127.0.0.1:6379/0
CELERY_RESULT_BACKEND=redis://127.0.0.1:6379/0
```

### 2.3 重启服务 (按顺序)

```bash
# 1. 重启 FastAPI (uvicorn/gunicorn) — 加载新代码
systemctl restart ihui-api    # 或: supervisorctl restart ihui-api

# 2. 重启 Celery worker (消费 run-reconcile 任务)
celery -A app.celery_app worker -l info -Q reconcile,default --detach
# 或: supervisorctl restart celery-worker

# 3. 启动 Celery beat (调度对账) — 进程级单例, 仅一个
celery -A app.celery_app beat -l info --detach
# 或: supervisorctl start celery-beat
```

⚠️ **注意**: beat 必须且只能有 1 个进程, 否则会重复触发对账。

### 2.4 部署后冒烟 (P0)

```bash
# 在 server 目录下
python scripts/verify_production_smoke.py
# 期望: 所有项 OK
```

### 2.5 一键运维流程 (按顺序执行)


## 3. 监控点 (告警/看板)

### 3.1 关键指标

| 指标 | 来源 | 告警阈值 |
|------|------|----------|
| `message` 表未读数 (`/api/admin/migration/notify/unread-count`) | admin 后台红点 | > 50 持续 1h |

## 4. 回滚方案

### 4.1 站内信持久化回滚 (仅 047 索引)

```bash
cd server
alembic downgrade -1
# 撤销 047 引入的 3 个索引, message 表结构不变
# API 仍可用, 失去索引加速
```

### 4.2 完整回滚到内存方案 (应急, 不建议)

1. `alembic downgrade -1` 回滚 047
2. 还原 `app/api/admin_migration.py` 至上一版本 (git revert)
3. 重启 API
4. 数据: DB 中已写入的站内信**不会自动回到内存**, 若需保留历史可忽略此步

### 4.3 Celery 回滚

```bash
# 停止 beat
supervisorctl stop celery-beat

# 若对账任务异常, worker 仍可继续消费其他任务
# 完全停止: supervisorctl stop celery-worker
```

### 4.4 前端红点回滚

若 30s 轮询导致 API 压力过大, 临时关闭:
- 编辑 `client/src/composables/useNotifyBadge.ts`, 将 `_POLL_MS` 调大或注释 `_start()`
- 重新 `npm run build:web`

## 5. 风险与限制

| 风险 | 等级 | 缓解 |
|------|------|------|
| 多副本 admin 收件箱 UUID 不一致 | 中 | env 强制固定 UUID, 部署清单已要求 |
| Celery beat 多实例重复调度 | 高 | 部署清单明确仅 1 个 beat 进程 |
| message 表写入与 edu 业务共用 | 低 | type='system_notice' 隔离, 索引无影响 |
| 047 迁移在 SQLite 环境被跳过 | 低 | 生产为 PG, 不影响 |
| 1000 条容量淘汰机制 | 低 | 仅淘汰非置顶, error 告警保留 |
