## 1. 站内信持久化 (替代内存方案)

### 1.1 背景
- 旧实现: `_notify_queue: deque` 内存队列
- 问题: 进程重启/多 worker/多 pod 不共享, 数据丢失

### 1.2 新方案 (PostgreSQL 持久化)
- 复用 `message` 表 (edu 业务也用), `user_id = NOTIFY_RECIPIENT_UUID` (固定 admin UUID)
- 字段复用: `sender_id` 存 `level` (info/warn/error), `sender_name` 存 `source` (migration/reconcile/...)
- error 级别 `is_top=True` 自动置顶
- 容量控制: `NOTIFY_MAX=1000` (FIFO 淘汰非置顶)
- 大整数 ID 序列化为 str (避免前端 JS 大数溢出)

### 1.3 迁移: 047_notify_persist
- 文件: `g:\IHUI-AI\server\alembic\versions\047_notify_persist.py`
- 状态: ✅ 已应用 (`alembic head = 047_notify_persist`)
- 索引 (PG only):
  - `idx_msg_user_unread (user_id, is_read)` — 未读计数
  - `idx_msg_user_created (user_id, created_at DESC)` — 时间倒序
  - `idx_msg_user_type (user_id, type)` — 分类过滤
- SQLite 环境: 跳过 (环境检测 `bind.dialect.name != "postgresql"`)
