## 四、关键技术决策识别

### 4.1 架构重写决策

| 维度 | H 盘历史 | G 盘新项目 | 决策 |
|------|---------|----------|------|
| **前端框架** | Vue 2 + JS + Element UI | Vue 3 + TS + Composition API + Element Plus | ✅ 升级 |
| **后端框架** | Spring Cloud 微服务（22 个 ihui-ai-edu-*-service）| FastAPI 单体 + 模块化目录 | ✅ 重写 |
| **状态管理** | Vuex 3 | Pinia | ✅ 升级 |
| **路由** | Vue Router 3 | Vue Router 4 | ✅ 升级 |
| **构建工具** | Vue CLI / Webpack | Vite | ✅ 升级 |
| **包管理** | npm | pnpm | ✅ 升级 |
| **类型系统** | JavaScript | TypeScript | ✅ 升级 |
| **测试** | 无统一测试 | Vitest + 大量 .test.ts | ✅ 新增 |
| **小程序** | 微信原生 + Vue | 微信原生 + Vue（保留） | ✅ 保持 |
| **样式** | SCSS | SCSS（保持） | ✅ 保持 |

### 4.2 微服务 → 单体决策

**H 盘 22 个 Java 微服务**（Spring Cloud Alibaba）：
- Nacos 注册/配置中心
- Sentinel 限流熔断
- RocketMQ 消息队列
- Seata 分布式事务
- Feign 远程调用
- Spring Cloud Gateway

**G 盘 FastAPI 单体**：
- 无注册中心（直接进程内调用）
- 无独立限流组件
- 无消息队列
- 无分布式事务
- 无远程调用（直接 import）
- 无 API Gateway（FastAPI 自身路由）

**影响**: 横向扩展能力下降，但运维复杂度大幅降低，适合中小规模业务。需要注意**单点故障**和**冷启动性能**问题。

### 4.3 数据库决策

- H 盘 Java 使用 MyBatis-Plus + MySQL（多库）
- G 盘 Python 使用 SQLAlchemy + MySQL（统一库 `app.database`）
- 部分 Java Domain 缺失（已在 `java_missing_models.py` 中标注）

### 4.4 重构决策

- **Admin 端通用组件化**: 139 → 102 个 .vue，通过 `AdminTableV2` / `AdminEditDialog` / `AdminBatchEditDialog` 收敛
- **Client 端领域扩张**: 94 → 187（新增 Agent/AI/AIGC/Refund/Recharge/Vip/Payment/Wallet 等业务）
- **Service 层拆分**: 新增 `client/src/api/services/` 业务逻辑封装层
