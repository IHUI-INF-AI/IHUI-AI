# 一、配置 (env)

```env
# 总开关
DUAL_WRITE_ENABLED=true

# 主备盘 (写)
DUAL_WRITE_PRIMARY=G        # G 为主盘 (新 PG), H 备盘
                              # 切回 H 主: DUAL_WRITE_PRIMARY=H

# 读盘
DUAL_WRITE_READ_FROM=G      # G 优先读
DUAL_WRITE_READ_FROM=H      # H 优先读
DUAL_WRITE_READ_FROM=BOTH   # 双读对比, G 兜底 (上线初期推荐)

# 对账
DUAL_WRITE_RECONCILE=true   # 开启定时对账
```

# 二、调用方式

## 1. 写 - 双写装饰器

```python
from app.services.dual_write import dual_write

@dual_write(
    table_name="edu_member",
    h_op=lambda data: h_member_dao.insert(data),
    g_op=lambda data: create_edu_member(**data),
)
def create_member(data):
    pass
```

## 2. 读 - 双读

```python
from app.services.dual_write import dual_read

def get_member(member_id: str):
    def read_h(): return h_member_dao.get(member_id)
    def read_g(): return get_edu_member(member_id)
    return dual_read(h_reader=read_h, g_reader=read_g, deep_compare=True)
```

## 3. 对账

```python
from app.services.dual_write import full_reconcile, reconcile_table

# 全量
reports = full_reconcile()

# 单表
report = reconcile_table("t_member", h_session_factory, g_session_factory)
```

# 三、阶段切换

| 阶段 | DUAL_WRITE_ENABLED | DUAL_WRITE_PRIMARY | DUAL_WRITE_READ_FROM | 持续时间 |
|---|---|---|---|---|
| 阶段 0: 准备 | false | - | G | 1-2 周 |
| 阶段 1: 双写双读 | true | G | BOTH | 2-4 周 |
| 阶段 2: G 优先 | true | G | G | 4-8 周 |
| 阶段 3: G 单写 | true | G | G (H 仍同步) | 2-4 周 |
| 阶段 4: H 停写 | false | - | G | 永久 |

# 四、监控指标

1. **写入一致性**: G 写成功率, H 写成功率, 双写一致率
2. **读取差异**: H/G 读结果差异数 (按表统计)
3. **对账**: 定时任务结果 (行数差异 / 内容差异)
4. **延迟**: G 写完成到 H 写完成的时差

# 五、风险与回滚

- 风险: G 盘写入失败 → 业务失败
- 风险: H 盘同步失败 → H 数据陈旧, 但不影响 G 业务
- 回滚: 设置 `DUAL_WRITE_ENABLED=false` 立即停止双写
- 回滚: 紧急可设 `DUAL_WRITE_PRIMARY=H` 回退到 H 主
