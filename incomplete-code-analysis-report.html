1. 高严重度:运行时错误(必须修复)

1.1 legacy_local.py — 3 个端点调用即 500 崩溃

文件: server/app/api/v1/auth/legacy_local.py

问题: /register、/me、/change-password 三个端点函数体仅 raise NotImplementedError(...),调用即返回 HTTP 500。

注册状态: 已注册 main.py:334 挂载于 /api/auth 前缀,在 _legacy_routers 列表中硬导入(非 try/except)。

前端影响: 有风险 前端 auth.ts:72 有 fallback 到 /api/auth/register 的逻辑,api-white-list.ts:32 将其列为公共路径。如果前端走到 fallback,会收到 500 错误。

根因: 从 client/backend/api/auth_routes.py 迁移的残留,注释指向 .migration_backup/ 但从未接线到 server 的 SysUser 模型。

建议: 要么实现这 3 个端点(接线到 auth_service),要么从 _legacy_routers 列表中移除,避免暴露不可用的端点。
